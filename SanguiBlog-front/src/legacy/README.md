# legacy/ 目录说明

该目录用于存放**已弃用但暂时保留**的历史代码/资源，目的有二：

1. 降低误维护风险：避免后续开发者在 `src/` 主流程中误改、误引入旧组件。
2. 保留参考价值：必要时可对比旧实现，但不应作为现网构建入口的一部分。

## 使用规范

- **禁止在现网入口中引用** `src/legacy/**` 下的任何文件。
- 若发现某个文件已经长期无引用且无保留价值，建议直接删除（需要时可通过 Git 历史找回）。
- 若确实需要从 legacy 复用，请先确认与当前 `src/appfull/**` 架构一致，并在复用后把 legacy 版本删除，避免“双份实现”长期并存。

## 已迁入的内容（示例）

- `legacy/components/*`：早期原型组件（旧版首页列表 / Hero / 导航 / 评论 / 按钮 / 统计条），已由 `src/appfull/**` 与 `src/components/common/**` 替代。
- `legacy/assets/react.svg`：Vite 模板默认资源，项目未引用，保留仅供参考。
