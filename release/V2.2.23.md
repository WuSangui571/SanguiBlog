# SanguiBlog V2.2.23 发布说明

发布日期：2026-04-20  
版本性质：首页体验、后台运维与发布流程增强版  
适用范围：SanguiBlog 全站（首页文章区、访问日志、后台设置页、文章封面上传、后台导航、站点文档）

---

## 一、版本概览

`V2.2.23` 基于 `V2.2.20` 继续演进。本次更新的重点不再是前一阶段的安全收口，而是把近期几轮已经落地的首页体验优化、后台可观测能力和编辑流程护栏整理为一个更完整的对外版本。

这次版本可以概括为五条主线：

1. 首页进一步优化手机端阅读节奏，并通过文章区分阶段加载降低首屏卡顿。
2. 后台新增系统监控分组，开始支持面向部署主机的实时指标与历史流量概览。
3. 访问日志补齐旧站 301 跳转来源识别，便于运营侧判断外部导流效果。
4. 文章封面上传链路增加更强的前端护栏，减少“文件已落盘但文章未绑定封面”的发布事故。
5. 后台桌面端侧栏支持折叠为纯图标模式，在不影响手机端抽屉逻辑的前提下，为主内容区释放更多宽度。

---

## 二、重点更新

### 1. 首页手机端与首屏性能继续收敛

`V2.2.23` 延续了 `V2.2.21` 之后对首页的连续优化，但重点已经从单纯“视觉改造”转向“真实使用手感”。

首页文章区现在采用分阶段加载策略：首屏先稳定渲染 Hero 与占位骨架，再在空闲或接近可视区时挂载文章列表，并结合预取与首次查询跳过机制，减少点击“向下探索内容”后从一套加载态切到另一套加载态的顿挫感。这意味着首屏更轻，手机端也更容易快速落到真实内容。

围绕手机端文章列表，本版本还收敛了多处细节，包括：

- 文章卡片改为更紧凑的左右结构，封面与正文比例重新平衡。
- 标题与摘要在手机端统一控制为更稳定的两行节奏，避免第三行露半截。
- 搜索卡片改成“标题与统计同一行、输入框单独一行”的布局，减少小屏拥挤感。
- 页码下方的站点金句去掉额外双引号，并缩小手机端字号，避免默认文案被挤成两行。

这些调整不改变首页的接口与数据结构，主要提升阅读连贯性和小屏视觉稳定性。

### 2. 后台新增系统监控分组

后台 `/admin/settings` 现已增加“系统监控”分组，并被调整为设置页默认首项。这个能力的目标是帮助管理员直接在站内查看部署主机的核心运行状态，而不是只关注应用业务数据。

当前监控内容包括：

- 操作系统信息
- 项目运行时长
- CPU、系统内存、磁盘占用
- 网卡实时吞吐量
- 今日 / 近 7 天 / 全部记录 / 本次开机的网络总流量概览
- 综合健康评分

实现上优先复用了跨平台指标采集能力，并对“项目运行时长”与“服务器开机时长”做了语义区分，最终展示的是当前 Java 进程运行时间，更贴近项目运维视角。

如果部署库尚未执行新增的监控快照建表 SQL，系统会自动降级为只展示本次开机累计流量，不会因为缺表导致后台直接不可用。

### 3. 访问日志支持识别旧站 301 跳转来源

如果你有旧域名或旧站点通过 301 跳转把流量导入当前博客，后台 `/admin/analytics` 现在可以更可靠地识别这类来源。

新版本会优先解析 `sg_redirect_from` 或 `redirect_from` 查询参数，把来源展示为“来自重定向：<旧域名>”，并同时接入普通页面埋点与文章详情页埋点，减少“旧站已导流，但后台来源列仍看不出来”的情况。

这项能力适合以下场景：

- 老站迁移到新域名后的导流观察
- 临时活动页或外层跳转页的投放归因
- 访问日志中快速判断历史站点迁移流量

### 4. 文章封面上传链路增加发布护栏

文章封面上传此前已经做过后端链路修复，但在真实发布流程中，仍可能出现“封面文件已写入磁盘、文章却因为上传尚未完成而以空封面发布”的问题。

这次版本在前端补上了更明确的保护：

- 封面上传请求加入超时兜底，避免界面长期停留在“上传中”。
- 当封面仍在上传时，新建文章与编辑文章的发布/保存按钮会被禁止。
- 按钮文案会明确提示“封面上传中...”，减少误操作。

这使得封面上传、文章保存、首页封面展示三者之间的状态链路更一致，也能减少后台未引用图片中已有封面文件、但前台文章仍显示默认图的情况。

### 5. 后台桌面端侧栏支持折叠

后台 `/admin` 的桌面端左侧导航不再只能固定占用整块宽栏。现在可以通过顶部按钮切换折叠状态：

- 展开时保持原有“图标 + 文本”的完整导航形态。
- 折叠后左栏缩为更窄的纯图标模式，右侧主内容区同步获得更大可用空间。
- 再次点击可恢复原样。

这次调整特别处理了桌面端与手机端的职责分离：

- 桌面端使用独立的折叠状态，不复用手机端抽屉开关。
- 手机端不再显示这个桌面折叠按钮，仍保持原有侧拉菜单逻辑。

因此，这是一项“桌面端阅读密度优化”，而不是对移动端导航行为的改写。

## 三、对部署与运维的影响

本版本建议前后端同时部署。

需要特别注意：

1. 系统监控分组依赖后端监控接口与前端展示同时更新，单侧部署无法获得完整能力。
2. 若希望使用“今天 / 近 7 天 / 全部记录”这类历史流量统计，请同步执行最新 `sanguiblog_db.sql` 中与 `system_monitor_snapshots` 相关的建表语句。
3. 如果你需要在访问日志中识别旧站 301 来源，旧站或反向代理层需要显式追加 `sg_redirect_from` 或 `redirect_from` 参数；单纯裸 301 不一定能稳定保留来源信息。
4. 文章封面上传护栏主要是前端发布流程保护，若只更新后端，无法获得“上传中禁止发布/保存”和超时兜底提示。
5. 后台侧栏折叠属于前端壳层能力，若只更新后端，后台布局不会发生变化。

## 四、升级建议

如果你当前仍停留在 `V2.2.20`，建议升级到 `V2.2.23`，尤其是在以下场景下收益明显：

- 你希望首页手机端首屏更轻、文章区加载更顺滑。
- 你需要在后台直接查看部署主机的资源占用与网络流量概览。
- 你正在做旧站迁移，希望后台访问日志能识别重定向导流来源。
- 你经常在后台发布文章，希望降低封面上传未完成时误点发布的风险。
- 你希望后台桌面端在宽屏下展示更多正文内容，而不是长期被固定侧栏占据空间。

## 五、兼容性说明

- 本版本仍属于 `V2.2.x` 范围内的连续增强版本，不是破坏性重构。
- 现有 AI 助理、博客文章 RAG、当前页面上下文增强、后台 AI 审计、首页背景图后台管理等能力继续保留。
- 系统监控历史流量在缺少快照表时会自动降级，不会因为数据库未迁移而直接阻塞后台页面。
- 后台侧栏折叠仅针对桌面端生效；手机端仍保持原有抽屉式导航，不属于功能回退。
- 本文档基于 `release/V2.2.20.md` 之后的 AI 变更日志整理而成，重点保留用户可感知改动，未逐条展开纯内部重构或测试性留痕。
