# SanguiBlog V2.2.4 发布说明

> **版本性质**：功能补强与稳定性修复版（相对 `V2.2.3`）
> **文档生成日期**：2026-03-27
> **站点版本号来源**：后端 `SanguiBlog-server/src/main/resources/application.yaml` 的 `site.version`，前端首页 Banner 展示 `SANGUI BLOG // <version>`
> **说明**：`V2.2.4` 主要补齐了基于邀请码的公开注册闭环，新增超级管理员邀请码生成能力，并持续打磨注册流程的交互体验；更细粒度的开发记录请参考 `.ai/CHANGELOG_AI.md`。

## 1. 版本亮点

- **开放邀请码注册能力**：新增公开注册页，用户只有在邀请码校验通过后，才能继续完成账号注册。
- **超级管理员可生成邀请码**：在 `/admin/settings` 新增注册邀请码分组，支持按时效生成一次性邀请码。
- **邀请码具备时效与单次消费特性**：邀请码支持过期控制，且一个邀请码只能成功注册一个新用户。
- **注册流程交互持续优化**：邀请码验证冷却、步骤显隐、返回重新验证、头像预览等体验得到补强。
- **后台最近邀请码结果支持刷新后保留**：系统设置页可加载最近一次生成的邀请码结果，便于管理员复制和核对。

## 2. 主要变更（摘要）

### 2.1 邀请码注册闭环

- 新增 `/register` 注册页，注册流程改为两步式：
  - 第一步仅验证邀请码
  - 第二步填写头像、用户名、显示名称、密码、确认密码
- 新增公开接口用于邀请码校验与注册提交，注册成功后邀请码会被一次性消费。
- 注册用户默认创建为最低权限的普通用户，避免匿名注册入口暴露过多后台字段。

### 2.2 超级管理员邀请码生成

- `/admin/settings` 新增“注册邀请码”分组，仅超级管理员可用。
- 支持默认 `5分钟` 时效，并提供 `1小时`、`1天`、`10天` 等扩展选项。
- 生成前提供确认流程，生成后自动尝试复制邀请码到剪贴板并给出提示。
- 后台支持读取最近一次生成的邀请码结果，刷新页面后不会丢失最近记录。

### 2.3 注册页体验优化

- 邀请码验证冷却期间，按钮会进入禁用态并显示倒计时。
- Step1 验证通过后，页面只显示 Step2 注册表单，不再混合展示验证区。
- Step2 增加“返回重新验证”操作，便于管理员或用户重新更换邀请码。
- 注册头像选择体验增强，补上本地预览与异常兜底，降低“已选择但看不到”的困惑。

### 2.4 稳定性与展示同步

- 首页版本展示同步更新到 `V2.2.4`，确保前后端版本口径一致。
- 根目录 `README.md` 中与当前版本和 release 状态相关的说明已同步更新。

## 3. 数据库与接口变更

本版本包含 **数据库结构扩展**，请确保根目录 `sanguiblog_db.sql` 已同步到 `V2.2.4` 对应版本。

### 3.1 MySQL

新增注册邀请码表：

- `registration_invites`

该表用于持久化邀请码、过期时间、创建人、消费人和消费时间，支撑：

- 超级管理员生成邀请码
- 邀请码时效校验
- 邀请码一次性消费
- 后台最近邀请码结果读取

### 3.2 新增/调整接口

- 超级管理员接口：
  - `/api/admin/registration-invites`
  - `/api/admin/registration-invites/latest`
- 公开注册接口：
  - `/api/auth/register/invite/verify`
  - `/api/auth/register`

## 4. 升级与部署建议

1. **数据库**：执行最新的 `sanguiblog_db.sql`，确保 `registration_invites` 表已创建。
2. **后端**：重新构建并部署 `SanguiBlog-server`，确保公开注册接口与后台邀请码接口已生效。
3. **前端**：重新构建并部署 `SanguiBlog-front/dist`，确保 `/login`、`/register` 与 `/admin/settings` 的最新页面生效。
4. **权限校验**：确认超级管理员账号可以访问 `/admin/settings` 中的“注册邀请码”分组。
5. **站点验证**：确认首页 Banner 展示版本已更新为 `V2.2.4`。

## 5. 验证建议

- 访问首页，确认 Banner 版本号显示为 `V2.2.4`。
- 打开 `/login`，确认存在“去注册”入口。
- 进入 `/register`，确认未验证邀请码前只显示邀请码输入与验证。
- 使用有效邀请码验证，确认通过后才出现注册表单。
- 用同一个邀请码完成一次注册后再次验证，确认邀请码不可重复使用。
- 进入 `/admin/settings`，确认超级管理员可生成邀请码并看到最近一次生成结果。

## 6. 已知注意事项

- 当前注册链路已具备前端冷却提示，但若后续需要更强的抗刷能力，仍建议继续补服务端限流。
- 注册成功后当前默认回到登录页，不会自动登录，这是当前版本的预期行为。
- 若生产环境启用了额外网关、WAF 或 CDN，请确认公开注册相关接口没有被误拦截。
