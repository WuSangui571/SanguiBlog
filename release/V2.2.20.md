# SanguiBlog V2.2.20 发布说明

发布日期：2026-04-11  
版本性质：安全收口与前台体验深度收敛版  
适用范围：SanguiBlog 全站（后端鉴权/上传链路、前台首页、文章详情页、移动端体验、后台编辑器、站点文档）

---

## 一、版本概览

`V2.2.20` 基于 `V2.2.6` 持续演进。和 `V2.2.6` 主要围绕首页视觉、后台管理与 AI 助理基础能力不同，这一版更偏向“上线质量收口”：一方面补上了若干高优先级安全与权限边界问题，另一方面对文章详情页、移动端交互、首页动态节奏与全站玻璃化提示体系做了密集打磨。

这次版本可以概括为四条主线：

1. 修复头像删除链路越界风险，收紧文章资源上传权限，并修复用户名修改后 JWT 立即失效的问题。
2. 大幅收敛文章详情页体验，尤其是手机端按钮、目录、分享提示、大图预览、异常页与玻璃风统一。
3. 持续优化首页与移动端体验，包括首页 Hero 滚动、手机端首篇文章跳转、搜索文案、卡片 hover 策略与彩蛋提示风格统一。
4. 补齐后台编辑器与前台若干深色模式细节，减少视觉割裂与使用阻力。

---

## 二、重点更新

### 1. 安全与权限边界补强

`V2.2.20` 首先处理了几项更偏“上线安全质量”的问题。

本版本修复了头像更新链路中旧头像文件删除可能越界的问题：此前若旧头像路径被构造成异常相对路径，后续换头像时理论上存在误删头像目录外文件的风险；现在头像路径会重新规范化并校验到头像根目录内，删除逻辑不再直接信任历史 URL。

同时，文章封面与正文资源上传权限也完成了收口。此前 `/api/upload` 下与文章写作相关的上传接口只要求“已登录”，现在已按文章管理权限收紧，避免普通登录用户直接向公开 `/uploads/**` 目录写入文章资源。

此外，用户自行修改用户名后当前 JWT 立即失效的问题也已修复。现在鉴权链路不再只依赖旧用户名作为唯一查找依据，用户改名后不会出现“刚改完资料，刷新就掉线”的问题。

### 2. 文章详情页体验集中收敛

文章详情页是本轮打磨最密集的区域。

在移动端，文章页原先存在多个浮动入口带来的视觉干扰与错位问题。本版本已移除手机端的“首页”“评论”“目录”悬浮按钮及其目录抽屉，仅保留桌面端对应能力，避免小屏下内容层被漂浮控件反复打断。

分享复制提示也经历了完整重做：从旧式、突兀的“链接已复制”提示，升级为统一的玻璃浮层提示卡片，并逐步优化到当前版本的“偏上位置、无首帧延迟、保留玻璃质感、页面结构不抖动”的稳定状态。这套提示样式已沉淀为站内共享模板，后续新增类似短提示时可直接复用。

文章异常状态页也统一到了当前站点玻璃语言，包括：

- 文章加载失败页
- 文章 404 / 不存在页

另外，正文图片大图预览时被“首页按钮”与 AI 入口压层、右侧出现白线的问题也已修复。现在打开大图时，相关浮层会正确隐藏并禁点，预览层级和滚动条表现更稳定。

### 3. 首页与移动端交互持续打磨

首页仍然是 `V2.2.x` 的持续优化重点，但本轮从“视觉接近模板”逐渐转向“真实使用稳定性”。

在手机端，“向下探索内容”按钮已经不再依赖旧的状态条锚点，而是直接跳到第一篇文章位置，自动跳过文章搜索卡片，解决了手机端首屏落点不准的问题。

首页 Hero 文案滚动也做了多轮收敛：早期版本中，文案淡出过快、移动节奏过猛，后续又出现了“快速滚动时补偿位移错位”的问题。当前版本最终收敛为更接近参考模板的原生滚动节奏，保留缓慢淡出，同时减少快速滚动下的视觉错位感。

首页搜索卡片中的默认提示文案已缩短，更适合手机端展示；首页博主信息卡与全部标签卡的外层 hover 上浮也已取消，仅保留更需要的内部交互反馈，整体更克制。

### 4. 彩蛋提示与玻璃风格统一

本版本还完成了站内若干“旧风格彩蛋/提示”的统一收口。

首页博主头像的连点彩蛋、主题切换按钮的“超频模式”提示，现在都已经接入站点统一的玻璃提示体系，不再沿用旧的黑边赛博弹层；同时修复了主题超频提示曾因变量引用错误导致前端白屏的问题。

这意味着站点前台目前已经形成一套相对统一的“非阻塞短提示”风格：文章分享提示、头像彩蛋提示、主题超频提示等，都尽量复用同一视觉语言和模板。

### 5. 深色模式与编辑器细节适配

除了前台交互，本次也补齐了一些长文阅读与后台写作场景中的深色模式细节。

包括：

- 文章页代码块滚动条的黑夜模式适配
- 后台新建文章页与编辑文章页中 Markdown 正文区域滚动条的黑夜模式适配

这些变化不属于“功能新增”，但会显著减少长文写作和夜间阅读时的违和感。

## 三、对部署与运维的影响

本版本建议前后端同时部署，不建议只替换单侧资源。

需要特别注意：

1. 后端安全与权限收口涉及头像处理、上传权限和 JWT 鉴权兼容，若只更新前端无法完整生效。
2. 前端文章详情页、首页 Hero、手机端交互和玻璃提示模板有较多细节调整，若只更新后端则无法获得对应体验修复。
3. 本版本不要求新增数据库结构迁移脚本，但建议确保你使用的是当前仓库最新 SQL 与代码版本组合。
4. 后端仍然通过 `application.yaml` 引入 `application-local.yaml`；其中：
   - `application.yaml` 应提交到 Git
   - `application-local.yaml` 不应提交到 Git，继续仅用于私有配置

## 四、升级建议

如果你当前仍停留在 `V2.2.6` 或更早的 `V2.2.x`，建议升级到 `V2.2.20`，特别是在以下场景下收益明显：

- 你关注站点上线安全，希望先修复头像删除、上传权限和改名后 JWT 失效等问题
- 你已经启用了文章页分享、图片预览或移动端阅读，希望减少按钮干扰与视觉错位
- 你正在持续打磨首页质感，希望获得更稳定的 Hero 滚动与更统一的玻璃提示风格
- 你经常在暗色模式下写文章或读长文，希望补齐滚动条与异常页的视觉一致性

## 五、兼容性说明

- 本版本仍属于 `V2.2.x` 范围内的连续演进，不是破坏性重构版本。
- 现有 AI 助理、RAG、首页背景图后台管理、历史会话、站点地图等能力继续保留。
- 手机端文章页移除了若干悬浮入口，这是有意的交互收敛，不是功能丢失；桌面端相关入口仍保留。
- 若你需要进一步了解每一次小版本迭代的细节，可继续参考仓库内 `.ai/CHANGELOG_AI.md`。
